# Concepts
Image — blueprint for a container (the how to build envrionement)
Container — running instance of an image (running environment)
Dockerfile — instructions to build an image
docker-compose — run and configure multiple containers

# Common Commands
docker build -t [image] .         # build image
docker run --name [container name] [image]                # run container
docker run -it [image] bash       # interactive shell
docker ps                         # list running containers
docker stop container_id          # stop container
docker-compose up                 # start all services
docker-compose down               # stop all services
docker images                     # list images
docker rmi [image]                # delete image

# Dockerfile Commands
FROM        # base image
WORKDIR     # working directory inside container
COPY        # copy files into container
RUN         # execute command at build time
CMD         # command to run when container starts
ENV         # set environment variables
EXPOSE      # document which port the container uses

# Reuse Images
docker run [image] python [script].py
FROM my-image

# Ect
containers stop after once all processes are done

# Instructions
Create Image with Docker files
Run scripts with docker run [image] [cmd]

use docker-compose up
docker exec -it [container_name] [cmd]
* container name set during creation or within compose file
